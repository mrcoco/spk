Tentang Metode SAW

Simple Additive Weighting (SAW) adalah metode yang digunakan untuk mengklasifikasikan peluang kelulusan mahasiswa dengan mempertimbangkan bobot dari setiap kriteria:

  • IPK (35%) - Benefit: Semakin tinggi semakin baik
  • Total SKS (37.5%) - Benefit: Semakin tinggi semakin baik
  • Nilai D/E/K (37.5%) - Cost: Semakin rendah semakin baik

Metode ini menghitung nilai preferensi untuk setiap alternatif dengan mengalikan nilai ternormalisasi setiap kriteria dengan bobotnya.

Threshold Klasifikasi:

  • Skor ≥ 0.7 = Peluang Lulus Tinggi
  • 0.45 ≤ Skor < 0.7 = Peluang Lulus Sedang
  • Skor < 0.45 = Peluang Lulus Kecil

Tentang Evaluasi

Evaluasi ini mengukur seberapa baik metode FIS dalam mengklasifikasikan peluang kelulusan mahasiswa. Semakin tinggi nilai accuracy, precision, recall, dan F1-score, semakin baik performa model.

Metrik Evaluasi

  • Accuracy: Proporsi prediksi yang benar dari total prediksi
  • Precision: Proporsi prediksi positif yang benar
  • Recall: Proporsi kasus positif yang berhasil diprediksi
  • F1-Score: Rata-rata harmonik dari precision dan recall

Catatan Penting

Hasil evaluasi ini menggunakan ground truth yang dibuat berdasarkan kriteria sederhana. Untuk evaluasi yang lebih akurat, diperlukan data historis kelulusan mahasiswa yang sebenarnya.

Proses Penggunaan Aplikasi

  1. Login: Pengguna masuk ke aplikasi menggunakan akun yang telah terdaftar. Keamanan data dijaga dengan sistem autentikasi dan hak akses sesuai role.
  2. Input Data: Pengguna menambahkan atau memperbarui data mahasiswa dan nilai. Data yang lengkap dan valid menjadi fondasi utama untuk proses selanjutnya.
  3. Klasifikasi: Sistem melakukan klasifikasi peluang lulus mahasiswa berdasarkan data yang telah diinput. Proses ini menggunakan metode SAW atau FIS sesuai pilihan pengguna.
  4. Evaluasi: Setelah klasifikasi, pengguna dapat menjalankan evaluasi untuk melihat performa model. Sistem akan menampilkan confusion matrix, metrik akurasi, precision, recall, dan F1-Score secara otomatis.
  5. Hasil & Analisis: Hasil evaluasi divisualisasikan dalam bentuk tabel, grafik, dan narasi otomatis. Pengguna dapat dengan mudah memahami kekuatan dan kelemahan model dari visualisasi ini.
  6. Rekomendasi: Sistem memberikan saran perbaikan dan ringkasan performa berdasarkan hasil analisis. Pengguna dapat mengikuti rekomendasi untuk meningkatkan kualitas data dan akurasi model.
Tips: Lakukan evaluasi ulang setiap kali ada perubahan data untuk mendapatkan hasil yang paling akurat dan rekomendasi terbaru.

  • Ringkasan Performa: Narasi otomatis tentang kekuatan & kelemahan model.

Keamanan & Privasi Data

  • Data mahasiswa dan nilai hanya dapat diakses oleh user yang berwenang.
  • Selalu logout setelah selesai menggunakan aplikasi, terutama di komputer bersama.
  • Jangan membagikan password ke siapapun, termasuk admin.
  • Backup data secara berkala untuk menghindari kehilangan data.

Rekomendasi & Ringkasan

  • Baca bagian rekomendasi untuk saran perbaikan spesifik (misal: perbaiki data IPK, cek SKS, validasi prodi).
  • Ringkasan performa membantu memahami hasil secara cepat dan mengambil keputusan.
  • Ikuti rekomendasi untuk meningkatkan akurasi model dan kualitas data.
Best Practice: Lakukan evaluasi ulang setelah melakukan perbaikan data.

Troubleshooting & FAQ

  • Data tidak muncul? Cek koneksi, refresh halaman, pastikan data sudah diinput.
  • Evaluasi gagal? Pastikan semua data wajib sudah terisi dan valid.
  • Tampilan tidak sesuai? Lakukan hard refresh (Ctrl+Shift+R/Cmd+Shift+R).
  • Bagaimana reset password? Hubungi admin/operator untuk reset password.
  • Bagaimana menambah prodi baru? Gunakan menu Program Studi, klik Tambah Program Studi.
  • Siapa yang bisa akses evaluasi? Semua user yang punya hak akses sesuai role.
  • Kontak bantuan: admin@email.com / WhatsApp Admin
Catatan: Jika menemukan bug/error, segera laporkan ke admin/operator.

Tentang Aplikasi

SPK Monitoring Masa Studi adalah aplikasi modern berbasis SPA untuk evaluasi kelulusan mahasiswa secara otomatis dan komprehensif.

Konsep & Tujuan

  • Membantu institusi memonitor masa studi dan peluang kelulusan mahasiswa secara efisien.
  • Memberikan evaluasi otomatis berbasis data real dan metode ilmiah (SAW & FIS).
  • Menyediakan rekomendasi dan insight berbasis analisis data.

Teknologi & Arsitektur

  • SPA (Single Page Application) dengan JavaScript, jQuery, dan Kendo UI.
  • Frontend: HTML5, CSS3, Bootstrap, FontAwesome, Chart.js.
  • Backend: Python FastAPI, PostgreSQL, Dockerized.
  • API: RESTful, endpoint terstandarisasi, dokumentasi otomatis.
  • Database: PostgreSQL, migrasi & backup terintegrasi.
  • CI/CD: Docker Compose, pipeline build & deploy otomatis.
Integrasi: Semua komponen saling terhubung melalui API dan environment yang terisolasi.

Mekanisme Pengujian

  • Unit Test: Pengujian fungsi backend (Python) dan komponen frontend (JS).
  • Integrasi: Pengujian alur data dari input hingga evaluasi hasil.
  • Manual Test: Uji coba fitur oleh user/operator dengan skenario nyata.
  • Data Real: Pengujian menggunakan data mahasiswa dan nilai asli untuk validasi akurasi.
Best Practice: Setiap perubahan kode diuji di lingkungan staging sebelum ke production.

Tim & Kontribusi

  • Pengembangan kolaboratif antara tim backend, frontend, dan data science.
  • Kontribusi terbuka untuk pengembangan fitur baru dan perbaikan bug.
  • Dokumentasi lengkap untuk memudahkan onboarding anggota baru.
